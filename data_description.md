# DATA DESCRIBTION

## AIGEN ##
- Human-LLM generated phishing x legitimate emails
- descr: his dataset includes a multiclass email dataset of 4000 Human-generated and LLM-generated emails of both phishing and genuine nature.
The Human-generated emails were taken by taking the most recent emails from Nazario and Nigerian Fraud datasets (1000 legitimate + 1000 phishing).
The LLM-generated emails were generated by ChatGPT and WormGPT (1000 legitimate + 1000 phishing). 
- ballanced 
- source: https://www.kaggle.com/datasets/francescogreco97/human-llm-generated-phishing-legitimate-emails/data

## ENRON ##
- safe x phishing
- descr: The Enron-Spam dataset is a fantastic ressource collected by V. Metsis, I. Androutsopoulos and G. Paliouras and described in their publication "Spam Filtering with Naive Bayes - Which Naive Bayes?". The dataset contains a total of 17.171 spam and 16.545 non-spam ("ham") e-mail messages (33.716 e-mails total).
- balanced
- source: https://github.com/MWiechmann/enron_spam_data?tab=readme-ov-file

## TREC ##
- safe x phishing
- descr: A preprocessed TREC 2007 Public Corpus Dataset suitable for building Spam Detection Models. The original dataset is from https://plg.uwaterloo.ca/~gvcormac/treccorpus07/about.html, here I just preprocessed the data so that it can be used simply. TREC 2007 Public Corpus Dataset is an email spam detection email. It contains 50199 spam emails and 25220 ham (not spam) emails.
- unbalanced
source: https://github.com/imdeepmind/Preprocessed-TREC-2007-Public-Corpus-Dataset?tab=readme-ov-file
